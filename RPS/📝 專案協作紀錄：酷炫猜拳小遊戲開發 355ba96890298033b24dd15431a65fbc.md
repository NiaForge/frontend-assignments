# 📝 專案協作紀錄：酷炫猜拳小遊戲開發

建立時間: May 3, 2026 11:02 PM

### **開發目標：** 製作一個符合課程邏輯要求，且具備現代視覺美感與互動體驗的 JavaScript 猜拳遊戲。

### 第一階段：基礎邏輯與視覺建立

- 
    
    **使用者需求：** 擔任前端設計師角色，要求根據作業規範（含 `getComputerChoice`、`judge`、`playGame` 三個函式及連勝紀錄功能）建立一個單一 HTML 檔案的酷炫遊戲 。
    
- **AI 執行重點：**
    - 
        
        **封裝邏輯：** 嚴格遵循教師提示，將程式碼模組化 。
        
    - 
        
        **視覺設計：** 導入 **Glassmorphism（玻璃擬態）** 風格，並使用 CSS 動畫增加動態感 。
        
    - 
        
        **技術應用：** 使用 `Math.random()` 處理隨機數，並以 **Template Literals** 優化字串輸出 。
        

---

### 第二階段：深度互動與遊戲性優化（關鍵轉折）

- 
    
    **使用者回饋與進階需求：** * **懸念感：** 提出增加「未知卡牌」的概念，電腦出拳不應直接顯示，而是增加「翻牌」動作 。
    
    - 
        
        **賽制優化：** 提議將單次勝負改為更具競技感的「三戰兩勝制」，並要求視覺化的統計呈現與最終勝利特效 。
        
- **AI 執行重點：**
    - 
        
        **翻牌特效：** 利用 CSS `transform: rotateY(180deg)` 實現 3D 翻牌動畫 。
        
    - 
        
        **計分系統：** 新增 `score-dot` 圓點指示燈，以綠色（勝）與紅色（負）即時視覺化賽果 。
        
    - 
        
        **狀態控制：** 加入按鈕鎖定機制，確保玩家在動畫播放期間無法重複點擊，強化回合制的節奏感 。
        
    - 
        
        **全局判斷：** 修改邏輯以判斷 `playerWins` 或 `computerWins` 是否先達兩勝，並觸發全螢幕遮罩（Overlay）顯示最終結果 。
        

---

### 💡 學習心得與 AI 使用習慣觀察

在這場協作中，我們展示了如何透過「迭代開發（Iterative Development）」來提升作品品質：

1. **從基礎到進階：** 先確保核心邏輯（作業要求）正確，再針對使用者體驗進行視覺與互動的堆疊。
2. **明確的意圖引導：** 使用者能準確描述「想要圖片翻牌感」、「三戰兩勝計分」等具體需求，讓 AI 能快速精準地修改程式碼。
3. **邏輯與美學並重：** 透過 AI 輔助處理繁瑣的 CSS 樣式與基礎邏輯，開發者可以將更多精力放在「如何讓遊戲更好玩」的創意決策上。
